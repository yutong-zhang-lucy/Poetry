# Product Blueprint: Ancient Poetry Mastery

> **Vision:** To bridge the gap between classical aesthetics and modern AI, creating a seamless "Search-Read-Recite-Record" ecosystem.

---

## 1. Problem Statement & Mission
We identified several friction points in the current poetry learning experience:
* **Fragmented Data:** Pinyin, annotations, and translations are often disconnected across multiple sources.
* **Passive Learning:** Users lack tools to actively track progress or memorize effectively.
* **Memory Blocks:** Traditional apps offer no "buffer" when a user gets stuck during recitation.

## 2. The Innovation: Progressive Recall Engine
The heart of our app is a **4-level dynamic hint system** designed to trigger active recall without spoiling the answer:

| Level | Mode | Visual Hint | Purpose |
| :--- | :--- | :--- | :--- |
| **L1** | **Seed** | First character only | Trigger initial memory. |
| **L2** | **Anchor** | First two characters | Establish phrase connection. |
| **L3** | **Imagery** | Semantic/Meaning hint | Contextual recall via AI. |
| **L4** | **Reveal** | Full sentence display | Final verification. |

## 3. Core Feature Set (MVP)
* **[ ] Multi-Layered Viewer:** Toggleable Pinyin (above characters) and Official Annotations (below lines).
* **[ ] Personal Archive:** Attach "Memory Hooks" or custom insights to any specific verse.
* **[ ] Mastery Tracking:** A dedicated dashboard for "Mastered" poems and recitation stats.
